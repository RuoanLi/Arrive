# Arrive — 搬迁与旅行规划助手

> 从落地那一刻起，清楚知道该做什么。

**Arrive** 将行程信息——目的地、停留时长、国籍、年龄和出行目的——转化为按优先级排列的逐日行动计划。它不提供通用清单，而是依据**紧急-重要矩阵（艾森豪威尔矩阵）**对任务进行排序，并将每项任务安排在相对于抵达和出发日期的具体时间节点上。

基于**原生 JavaScript、HTML 和 CSS** 构建的单页应用——无框架、无构建步骤、无外部依赖。

## ✨ 功能特性

- **个性化任务生成** — 根据出行目的（旅游、工作、学习、永久迁居）以及用户选择的可选"套餐"（宠物、药物、驾驶、银行、住房、家庭/受抚养人、高价值物品、自由职业）筛选任务。
- **艾森豪威尔矩阵视图** — 每项任务根据紧迫性和重要性被归入四个象限之一：*立即处理*、*计划安排*、*委托/自动化*、*减少/可选*。
- **优先级评分** — 通过加权评分（`重要性 × 2 + 紧迫性 + 时间加成`）对任务排序，最关键、最紧迫的事项优先显示。
- **动态时间轴** — 将象征性时间（如"出发前 14 天"、"第 1 天"、"第 2 周"）转换为实际天数，按行程时长和目的进行缩放，并归组为按时间顺序排列的清单。
- **情境感知规则** — 未成年人自动添加监护人同意文件（最高优先级），输入的国籍会生成针对性的护照/入境规则任务。
- **响应式设计** — 使用 `clamp()` 实现流体排版，CSS Grid 布局，以及移动端断点适配。

## ⚙️ 工作原理

核心逻辑位于 `app.js`，在浏览器中提交表单时完全在客户端运行。

1. **筛选** — `buildPlan()` 通过匹配所选目的和套餐，从任务模板库中选取相关任务。
2. **时间映射** — `mapDay()` 将每项任务的象征性时间转换为数字天数，根据实际行程时长对提前量和里程碑进行缩放。
3. **分类** — `pickQuadrant()` 根据 `urgency` 和 `importance` 值将每项任务分配到艾森豪威尔象限。
4. **评分与排序** — 每项任务获得一个优先级分数（截止日期越早，通过 `urgencyBonus()` 获得加成），按分数从高到低排序。
5. **渲染** — 使用 DOM API 和 HTML `<template>` 元素将结果渲染到矩阵和按天分组的时间轴中。

任务库完全由数据驱动：添加新任务只需在 `taskTemplates` 数组中追加一个对象——无需修改 UI。

## 🚀 快速开始

无需构建工具——纯静态文件。

### 方式一：直接打开

下载或克隆仓库，在任意现代浏览器中打开 `index.html`。

```bash
git clone https://github.com/RuoanLi/arrive-relocation-planner.git
cd arrive-relocation-planner
open index.html      # macOS
# 或直接双击 index.html
```

### 方式二：运行本地服务器（推荐）

本地服务器可避免浏览器兼容性问题，并与部署环境保持一致。

```bash
# Python 3
python -m http.server 8000

# 或 Node
npx serve
```

访问 `http://localhost:8000`。
